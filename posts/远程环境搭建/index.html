在踩完wsl2，mac mini的坑后，搞了一个ubuntu的mini主机，作为编译与部署服务器，由于桌面空间有限，只接网线与电源，日常使用ssh与samba就够用了，不过某些应用需要在图形界面下进行测试，专门购入一个毫米波的无线触屏显示器，无奈干扰和发热严重，最后还是选择使用远程桌面（在这个万物互联理念都推出了这么久了，还在用最古老的方法。。。心累

1. window2ubuntu
#

window2ubuntu选择xrdp就行：市面上虽然有更成熟的产品，却基本需要图形界面配置（完美循环依赖

sudo apt-get install xrdp

在注释文件中添加配置：

#/etc/xrdp/startwm.sh -> 解决本地登录黑屏
unset DBUS_SESSION_BUS_ADDRESS
unset XDG_RUNTIME_DIR

#~/.xsessionrc -> 同步桌面
export GNOME_SHELL_SESSION_MODE=ubuntu
export XDG_CURRENT_DESKTOP=ubuntu:GNOME
export XDG_CONFIG_DIRS=/etc/xdg/xdg-ubuntu:/etc/xdg

# 减少画面卡顿，作用有限
#/etc/xrdp/xrdp.ini
tcp_send_buffer_bytes=4194304
tcp_recv_buffer_bytes=6291456

#/etc/sysctl.conf
net.core.rmem_max = 12582912
net.core.wmem_max = 8388608

最后执行：

sudo sysctl -p
sudo systemctl restart xrdp

systemctl 管理命令：

sudo systemctl enable xrdp.service   # 开机启动，默认
sudo systemctl start xrdp.service    # 启动，安装后自动启动
sudo systemctl restart xrdp.service  # 重启
sudo systemctl stop xrdp.service     # 关闭
